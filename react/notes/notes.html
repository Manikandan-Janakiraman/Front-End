1. Difference — HTML Webpage & React Webpage

Quick comparison table

Aspect HTML Webpage React Webpage
Structure Built using static HTML, CSS, JS files. Built using components (JSX + JS) that render HTML dynamically.
Rendering Browser loads and renders full HTML each time. Uses Virtual DOM — only updates parts that change.
Reusability HTML elements are static; no modular reusability. React components are reusable and modular.
Interactivity Needs JavaScript manually for dynamic actions. React handles interactivity with state & props.
Performance Slower when reloading pages. Faster updates — Single Page Application (SPA).
Data Handling No built-in data management. Handles dynamic data easily using state management.
HTML → Static & simple React → Dynamic, component-based, SPA framework

3. What is TypeScript?

TypeScript is a superset of JavaScript that adds static typing.

let name: string = "Mani";
let age: number = 25;
// If you write: age = "twenty" — TypeScript will throw an error.
  • Type safety (detects errors early)
  • Better autocompletion & IntelliSense
  • Easier debugging
  • Used in frameworks like Angular and in React projects as TSX

4. Key JavaScript Concepts

Short definitions + examples

Variable

Container for storing data. let name = "Lunar";

Declaration

When you declare a variable: let x;

Initialization

Assign value: x = 10;

Lexical Scope

Variable access depends on where it’s declared in the code structure (not where it’s called).

Block Scope

Variable available only inside {}. let and const follow block scope.

Function Scope

Variables declared inside a function are accessible only within that function. (var uses function scope.)

Hoisting

JS moves declarations (not initializations) to top of scope. Example: you can use a var before declaring it.

Closure
function outer() {
  let count = 0;
  return function inner() {
    count++;
    console.log(count);
  };
}
const counter = outer();
counter(); // 1
counter(); // 2

5. Why Node.js in Web Development?

Node.js allows JavaScript to run on the server-side.

  • Handles backend logic, APIs, and databases.
  • Enables full-stack JavaScript (React frontend + Node backend).
  • Fast — built on V8 engine (Google Chrome’s JS engine).
  • Great for real-time apps (chat, notifications).

In short: Node = Backend environment for JavaScript.
